### 3.2 通用响应结构

后端常见响应结构如下（HTTP 状态码不总是用于区分业务错误）：

```json
{
  "code": 0,
  "msg": "请求成功",
  "data": {}
}
```

- `code === 0`：成功
- `code !== 0`：业务错误（移动端的请求层会原样返回）
- `code === 4008`：Token 无效或已过期（移动端会清 token 并跳转登录）
  - 见：[api.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-M/src/services/api.js#L153-L174) 与 [mobile/auth.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/middlewares/mobile/auth.js#L1-L40)

## 4. JWT 鉴权（移动端）

### 4.1 Token 生成

移动端注册/登录成功后返回 `token`，由后端使用 `JWT_SECRET` 签名生成：

- 生成位置：[services/mobile/auth.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/services/mobile/auth.js#L45-L210)
- Payload 结构（移动端）：
  - `user_id`
  - `phone`
  - `role`
- 过期时间：
  - 默认 `JWT_EXPIRES_IN`，不存在则 `2h`
  - 也支持前端传 `token_expires_in`（秒）覆盖

### 4.2 Token 传递方式

需要鉴权的接口统一在请求头加入：

```http
Authorization: Bearer <token>
```

移动端请求层会自动从本地存储读取 `token` 并注入 Header：
- 见：[api.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-M/src/services/api.js#L100-L105)

### 4.3 后端校验方式与失败行为

后端中间件会解析 `Authorization`，校验 JWT 并写入 `req.user`：
- 见：[mobile/auth.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/middlewares/mobile/auth.js#L1-L40)

校验失败时返回：

```json
{ "code": 4008, "msg": "Token 无效或已过期", "data": null }
```

移动端收到 `4008` 会清理登录态并跳转登录页：
- 见：[api.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-M/src/services/api.js#L153-L169)

### 5.2 后端处理链路（概览）

通常流程为：

1. 校验手机号格式（正则/validator）
2. 频率限制（通常 60s 内只允许发送一次）
3. 生成 6 位验证码并落库（`verification_codes`）
4. 调用阿里云短信服务下发验证码
5. 返回 `expires_in`

验证码生成与持久化涉及：
- 验证码生成：[code.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/utils/code.js#L1-L16)

### 6.2 /completion 请求与响应约定

```json

  "messages": [
    { "role": "user", "content": "帮我推荐上海的酒店" }
  ],
  "history": [
    { "role": "user", "content": "你好" },
    { "role": "assistant", "content": "您好，需要什么帮助？" }
  ]
}
```

响应体（后端会统一包装为 `code=0`，即使内部异常也尽量返回“友好消息”）：

```json
{
  "code": 0,
  "msg": "请求成功",
  "data": {
    "message": { "role": "assistant", "content": "..." },
    "tool_calls": []
  }
}
```

控制器实现参考：[chatController.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/agent/controllers/chatController.js#L1-L174)

3. 工具调用目前提供 `hotel_search`（查酒店/详情/对比），基于数据库检索

核心代码位置：
- 调度器：[customerAgent.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-backend/src/agent/core/customerAgent.js#L1-L203)

### 8.4 API 服务层与核心接口（前端侧）

移动端 API 请求统一封装在 [api.js](file:///d:/github/Yisu-Hotel/M/Yisu-Hotel-M/src/services/api.js#L1-L561)：

- BaseURL：当前写死为 `http://localhost:3001`
- Token：若本地存在 `token`，会自动注入 `Authorization: Bearer <token>`
- 错误处理：后端返回 `code===4008` 时会清理登录态与缓存，并跳转登录页
- GET 缓存：GET 请求基于 URL + body 生成 cacheKey 做本地缓存（默认 5 分钟）

核心接口清单（以 `src/services/api.js` 为准）：

- 城市
  - `GET /mobile/city/list`：`cityApi.getCities()`（当前页面实现中未直接使用）
- 认证
  - `POST /mobile/auth/login`：`authApi.login(credentials)`（登录页）
  - `POST /mobile/auth/register`：`authApi.register(data)`（注册页）
  - `POST /mobile/auth/send-code`：`authApi.sendCode(phone, type)`（注册页验证码）
- 用户
  - `GET /mobile/user/profile`：`userApi.getProfile()`（我的页）
- 广告
  - `GET /mobile/banner/list`：`bannerApi.getBanners()`（首页 banner）
- 酒店/收藏
  - `GET /mobile/hotel/list?query`：`hotelApi.getHotelList(params)`（首页/酒店列表页）
  - `GET /mobile/hotel/:hotelId`：`hotelApi.getHotelDetail(hotelId, params)`（酒店详情/预订确认兜底）
  - `GET /mobile/favorite/list`：`favoriteApi.getFavorites()` / `hotelApi.getCollectedHotels()`（收藏页/酒店详情检查收藏）
  - `POST /mobile/favorite/add`：`favoriteApi.addFavorite(hotelId)` / `hotelApi.collectHotel(hotelId)`（酒店详情收藏）
  - `POST /mobile/favorite/remove`：`favoriteApi.removeFavorite(hotelId)` / `hotelApi.uncollectHotel(hotelId)`（酒店详情取消收藏/收藏页取消）
- 预订/订单（后端实际复用 booking 路由）
  - `POST /mobile/booking`：`bookingApi.createBooking(bookingData)`（预订确认页）
  - `GET /mobile/booking/list?...`：`bookingApi.getBookingList(params)` / `orderApi.getOrders(params)`（订单页/我的页待支付数量）
  - `GET /mobile/booking/detail/:id?...`：`bookingApi.getBookingDetail(id)` / `orderApi.getOrderDetail(id)`（支付页/订单详情页）
  - `POST /mobile/booking/cancel`：`bookingApi.cancelBooking(orderId)` / `orderApi.cancelOrder(orderId)`（订单页/订单详情页）
  - `POST /mobile/booking/pay`：`bookingApi.payBooking(paymentData)` / `orderApi.payOrder(orderId)`（支付页/订单详情页）
- 优惠券
  - `GET /mobile/coupon/list?...`：`couponApi.getCoupons(params)`（优惠券页/预订确认页/支付页）
  - `POST /mobile/coupon/receive`：`couponApi.receiveCoupon(couponId)`（优惠券页）
  - `POST /mobile/coupon/use`：`couponApi.useCoupon(couponId, bookingId)`（支付页）
  - `GET /mobile/coupon/history`：`couponApi.getCouponHistory()`（当前页面实现未直接使用）
- 浏览历史
  - `GET /mobile/history/list`：`historyApi.getHistory()`（当前历史页实现走本地 `browsingHistory`，未调用该接口）
- AI 助手/客服
  - `POST /mobile/chat/completion`：`aiApi.chat(messages)`（AI 助手页、客服中心页）
  - `GET /mobile/chat/info`：`aiApi.getInfo()`（当前页面实现未直接使用）
  - `GET /mobile/chat/health`：`aiApi.healthCheck()`（当前页面实现未直接使用）
- 帮助中心
  - `GET /mobile/help/center`：`helpApi.getHelpCenter()`（帮助中心页）
